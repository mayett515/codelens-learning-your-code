Choose a concept above to see how each language expresses it. The decorator pattern is, at heart, a function that takes a function and returns a new function — a higher-order transformation that wraps behavior around behavior.
Python
imperative · 1991
def timed(fn): def wrapper(*args, **kw): start = time.time() result = fn(*args, **kw) print(time.time() - start) return result return wrapper @timed def greet(name): return f"hello {name}"
Clojure
lisp · 2007
(defn timed [fn] (fn [& args] (let [start (System/currentTimeMillis) result (apply fn args)] (println (- (System/currentTimeMillis) start)) result))) (def greet (timed (fn [name] (str "hello " name))))
F#
ml-family · 2005
let timed fn = fun args -> let sw = Stopwatch.StartNew() let result = fn args printfn "%d" sw.ElapsedMilliseconds result let greet = timed (fun name -> sprintf "hello %s" name)
— The Ladder of Abstraction —
  highest      ┃   category theory:    a natural transformation between functors
               ┃                       ( "wrap behavior around behavior" )
               ┃
               ┃   type theory:        timed : (a → b) → (a → b)
               ┃                       ( a function that returns a function of the same shape )
               ┃
               ┃   functional idiom:   higher-order function / decorator / combinator
               ┃                       ( all three languages express this directly )
               ┃
               ┃   syntactic sugar:    @decorator     |  (def x (deco f))  |  let x = deco f
               ┃                       Python's @     |  Clojure's nesting  |  F#'s binding
               ┃
  lowest       ┃   machine code:       allocate closure, capture fn pointer, jump to wrapper
               ┃                       ( all three compile to roughly this )
