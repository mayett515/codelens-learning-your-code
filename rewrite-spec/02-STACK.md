# Stack

Every choice here is deliberate. Do not substitute without asking.

## Runtime

- **Expo SDK 52+** with the dev client (not Expo Go — we need `op-sqlite` and `react-native-mmkv` which require native modules).
- **React Native 0.76+** (new architecture / Fabric / TurboModules enabled).
- **TypeScript 5.x, strict mode**, no implicit any, `exactOptionalPropertyTypes`.
- **Hermes** as the JS engine.

## Navigation

- **Expo Router (file-based)**. Routes mirror the screen list in `05-SCREENS.md`.
- Hardware back button handled centrally via `BackHandler` and the router's `usePathname`.

## State

- **Zustand** for client/UI state (current screen state, current selection, mode flags).
- **TanStack Query (React Query) v5** for server-state-ish data: anything backed by SQLite. Mutations invalidate query keys. This is our reactive layer.
- **MMKV** (`react-native-mmkv`) for synchronous KV: API keys (encrypted), small settings, last-opened-project pointer. *Not* for vectors or large blobs.

## Persistence

- **op-sqlite** as the SQLite driver. JSI-backed, synchronous, fast.
- **sqlite-vec** extension compiled into op-sqlite for vector search. Use `vec0` virtual tables.
- **Drizzle ORM** for type-safe schema + queries. Schema lives in `src/db/schema.ts`. Migrations in `src/db/migrations/` generated by `drizzle-kit`.
- **One database file**, multiple tables. No second store. No JSON cache of vectors. SQLite is the single source of truth for everything except ephemeral UI state and API keys.

## Graph Rendering

- **Cytoscape.js + cytoscape-cxtmenu**, vendored under `assets/vendor/cytoscape/`.
- Rendered inside a `react-native-webview` with a single static HTML file (`assets/graph.html`) that loads the vendored libs.
- Communication via `postMessage` in both directions — strictly typed message protocol in `src/graph/messages.ts`.
- Why WebView: Cytoscape is the right tool, has no native RN port, and the app already proved this works on phone with cxtmenu touch-tuning. Do not try to find an "RN-native" graph library.

## AI / Network

- **`fetch`** with an `AbortController` per request. No SDK dependencies for OpenRouter / SiliconFlow — they are plain HTTP.
- **Single serialized queue** for chat completions. Implementation in `src/ai/queue.ts`. Provider cooldowns + bounded retry/backoff (port the logic from `12-ai-api.js`).
- **Embedding calls** go through the same queue with a separate priority lane.

## Other Libraries

- **`react-native-gesture-handler`** and **`react-native-reanimated`** — required peers, used for marker long-press / range-select gestures and any custom animations.
- **`expo-haptics`** — feedback on mark, save-as-learning, long-press triggers.
- **`expo-secure-store`** as the storage backend for API keys (MMKV holds the pointer, secure-store holds the secret). Or use MMKV's encryption — pick one and document it. Default: `expo-secure-store` for secrets.
- **`zod`** for parsing AI responses (concept extraction returns JSON; validate it).
- **`date-fns`** for any date formatting. No moment.

## What Is Explicitly NOT in the Stack

- **No Realm.** Live objects + reactive queries are magic we don't need. Drizzle + TanStack Query is more honest and equally ergonomic.
- **No WatermelonDB.** Lazy loading solves a problem we don't have at our scale.
- **No Redux / Redux Toolkit.** Zustand is enough.
- **No styled-components / emotion.** Use StyleSheet or NativeWind (Tailwind for RN). Pick one in phase 1 and stick to it.
- **No async-storage.** MMKV replaces it everywhere.
- **No CDN-loaded assets.** All vendored.
- **No Java / Kotlin written by us.** If a feature requires it, defer the feature.
