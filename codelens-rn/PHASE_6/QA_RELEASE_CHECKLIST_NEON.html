What The App Does (For New Testers)

Product In One Line

CodeLens is a local-first mobile app for learning code. You import code, chat about it, save useful chat bubbles as learning concepts, and search those concepts later using embeddings.

Why Phase 6 QA Matters

Phase 6 is all about data safety: backup export, restore, clear-all, and resilience against bad files or app interruption. This checklist verifies that user data survives real-world failures.

ProjectA code workspace in the app.
FileA source file inside a project.

General ChatChat without attached code context.
Section ChatChat tied to selected code.
Learning ConceptStructured knowledge extracted from chat.
Learning SessionA grouped save event for one or more concepts.

Guided Step-by-Step Runbook

Each step tells you where to do it, exactly what to do, and what PASS means.

Step 1 - Preflight
Purpose: Set the baseline so all later comparisons are meaningful.
Where: One physical Android + one physical iOS device.
  1. Use build from commit e58a317.
  2. Record device models and OS versions in header fields.
  3. Start a count table for: projects, files, chats, chat_messages, learning_sessions, concepts, concept_links, embeddings.
PASS if: both devices are ready and your baseline table exists before testing.
Evidence to capture: tester name, devices, commit hash, start time.
Step 2 - Create Golden Dataset (Device A)
Purpose: Create realistic data to prove backup and restore are complete.
Where: Device A (source device for export).
  1. Open Settings. Save both API keys (dummy values are fine).
  2. Change one chat model and one embedding model to non-default strings.
  3. Create 2 projects (one paste; one GitHub import path if available).
  4. Create at least 4 files total with marks/ranges.
  5. Create 1 general chat with 4+ messages.
  6. Create 1 section chat with 4+ messages.
  7. Save at least 3 chat bubbles as learning.
  8. Open Learning Hub; confirm Concepts and Sessions are non-zero.
  9. Open Dev screen; run Run RAG Smoke Test once.
PASS if: all listed entities exist and Dev smoke test succeeds once pre-export.
Evidence to capture: baseline counts and screenshots of Learning Hub non-zero totals.
Step 3 - Export + Tear Apart Archive
Purpose: Validate the exported backup file internals before restore.
Where: Export on Device A, inspect on desktop PowerShell.
  1. On Device A: Settings -> Data -> Export backup.
  2. Save the .codelens file to desktop.
  3. Run the audit script below in PowerShell (set $src first).
  4. Confirm metadata.magic = codelens-backup, formatVersion = 1, schemaVersion = 2.
  5. Confirm NDJSON counts match metadata counts.
  6. Confirm at least one embedding decodes to 1536 bytes.
$src = "C:\path\to\codelens-backup-YYYY-MM-DD.codelens"
$work = "$env:TEMP\codelens_audit"
if (Test-Path $work) { Remove-Item -Recurse -Force $work }
New-Item -ItemType Directory -Path $work | Out-Null
Copy-Item $src "$work\backup.zip"
Expand-Archive -LiteralPath "$work\backup.zip" -DestinationPath "$work\unzipped" -Force

$required = @(
  "metadata.json","preferences.json","secure_keys.json",
  "projects.ndjson","files.ndjson","chats.ndjson","chat_messages.ndjson",
  "learning_sessions.ndjson","concepts.ndjson","concept_links.ndjson"
)
$missing = $required | Where-Object { -not (Test-Path "$work\unzipped\$_") }
"Missing files: $($missing -join ', ')"

$meta = Get-Content "$work\unzipped\metadata.json" -Raw | ConvertFrom-Json
$meta | ConvertTo-Json -Depth 5

function Count-Ndjson([string]$p) {
  if (-not (Test-Path $p)) { return 0 }
  return ((Get-Content $p) | Where-Object { $_.Trim().Length -gt 0 }).Count
}
@{
  projects = Count-Ndjson "$work\unzipped\projects.ndjson"
  files = Count-Ndjson "$work\unzipped\files.ndjson"
  chats = Count-Ndjson "$work\unzipped\chats.ndjson"
  chat_messages = Count-Ndjson "$work\unzipped\chat_messages.ndjson"
  learning_sessions = Count-Ndjson "$work\unzipped\learning_sessions.ndjson"
  concepts = Count-Ndjson "$work\unzipped\concepts.ndjson"
  concept_links = Count-Ndjson "$work\unzipped\concept_links.ndjson"
} | ConvertTo-Json

node -e "const fs=require('fs');const p='$work\\\\unzipped\\\\concepts.ndjson';const lines=fs.readFileSync(p,'utf8').split('\\n').filter(Boolean).map(JSON.parse);const e=lines.find(x=>x.embedding);if(!e){console.log('No embedded concepts found');process.exit(0)};const len=Buffer.from(e.embedding.vectorBase64,'base64').length;console.log('First embedding bytes:',len,'(expect 1536)')"
PASS if: archive structure is complete and metadata/count/vector checks all match.
Evidence to capture: PowerShell output and metadata snapshot.
Step 4 - Fresh Install Restore (Critical)
Purpose: Prove a clean device can fully recover from backup.
Where: Android and iOS clean installs.
  1. Android: clear app storage or reinstall. iOS: uninstall + reinstall.
  2. Confirm app starts empty (No projects yet).
  3. Go to Settings -> Data -> Restore from backup.
  4. Select known-good .codelens file and complete restore.
  5. Validate Restore complete counts.
  6. Verify UI data is back: home projects, recent chats, learning concepts/sessions, chat histories.
  7. Verify API keys are not restored as secrets and missing providers are listed.
PASS if: restored data is complete and keys require re-entry by design.
Evidence to capture: restore modal screenshot + before/after home screen.
Step 5 - Clear-All Data Tests
Purpose: Validate destructive flow safety with both key-handling modes.
Where: Android and iOS.
  1. Run Clear all data with checkbox OFF, type DELETE.
  2. Confirm flash message All data cleared.
  3. Verify app data is gone and API keys are still present.
  4. Re-import backup.
  5. Run Clear all data with checkbox ON, type DELETE.
  6. Re-open Settings and verify key inputs are empty (keys wiped).
PASS if: both modes behave exactly as intended (preserve vs wipe keys).
Evidence to capture: settings key state after each clear path.
Step 6 - Failure Path Tests
Purpose: Ensure bad inputs and user cancellation fail safely without corruption.
Where: Android and iOS.
  1. Cancel document picker during restore.
  2. Restore random non-zip file.
  3. Restore zip missing metadata.json.
  4. Restore archive with bumped formatVersion (edit metadata to higher value).
  5. Restore corrupted NDJSON (truncate one line in concepts.ndjson).
PASS if: app never crashes, shows clear Restore failed: ... feedback, remains usable.
Evidence to capture: error toasts and app still running after each case.
Step 7 - Interruption / Kill Tests
Purpose: Validate recovery if restore is interrupted mid-flight.
Where: Android and iOS.
  1. Start restoring a larger archive and kill app during spinner.
  2. Re-open app immediately.
  3. Confirm no crash loop and app opens cleanly.
  4. Run restore again and confirm it still succeeds.
PASS if: app recovers and can restore successfully after forced interruption.
Evidence to capture: reopen state + successful retry restore.
Step 8 - Post-Restore Functional Smoke
Purpose: Confirm normal user behavior still works after all destructive tests.
Where: Android and iOS.
  1. Send one new general chat message.
  2. Save one bubble as learning.
  3. Open learning chat for that concept.
  4. Run Dev screen smoke test once.
PASS if: no runtime errors and retrieval/learning flows still function.
Evidence to capture: quick note of successful send, save-learning, and dev smoke.
